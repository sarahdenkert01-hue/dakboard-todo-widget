Follow these steps to get your own personal to-do list widget running!

Step 1: Create a Free Supabase Account

  1. Go to supabase.com
  2. Click "Start your project" and sign up (it's free!)
  3. Create a new organization (just use your name)
  4. Click "New Project"
  5. Give it a name like "my-todo-list"
  6. Create a database password (save this somewhere safe!)
  7. Choose a region close to you
  8. Click "Create new project" (takes 2-3 minutes)

Step 2: Set Up Your Database

  1. In your Supabase project, click "SQL Editor" in the left sidebar
  2. Click "New Query"
  3. Copy and paste this entire SQL code:
-- Create tasks table CREATE TABLE tasks ( id bigint PRIMARY KEY GENERATED ALWAYS AS IDENTITY, list_id bigint NOT NULL, text text NOT NULL, completed boolean DEFAULT false, subtasks jsonb DEFAULT '[]'::jsonb, created_at timestamp with time zone DEFAULT now(), position int DEFAULT 0 ); ALTER TABLE tasks ENABLE ROW LEVEL SECURITY; CREATE POLICY "Allow all operations" ON tasks FOR ALL USING (true) WITH CHECK (true); -- Create lists table CREATE TABLE lists ( id bigint PRIMARY KEY GENERATED ALWAYS AS IDENTITY, name text NOT NULL, position int DEFAULT 0, x_position int DEFAULT 0, y_position int DEFAULT 0, width int DEFAULT 250, created_at timestamp with time zone DEFAULT now() ); ALTER TABLE lists ENABLE ROW LEVEL SECURITY; CREATE POLICY "Enable all access for lists" ON lists FOR ALL USING (true) WITH CHECK (true);
  1. Click "Run" at the bottom right
  2. You should see "Success. No rows returned"

Step 3: Enable Real-Time Updates

  1. Click "Database" in the left sidebar
  2. Click "Replication"
  3. Find the "tasks" table and toggle it ON
  4. Find the "lists" table and toggle it ON
  5. Click "Save"

Step 4: Get Your API Credentials

  1. Click the gear icon ‚öôÔ∏è (Project Settings) in the left sidebar
  2. Click "API"
  3. You'll see two important things:
    • Project URL - looks like: https://xxxxx.supabase.co
    • anon/public key - a long string starting with "eyJ..."
  4. Keep this tab open - you'll need these in the next step!

Step 5: Update Your HTML Files

You need to add your credentials to BOTH index.html and mobile.html files:

  1. Open index.html in a text editor
  2. Find these lines near the top of the <script> section:
const SUPABASE_URL = 'YOUR_SUPABASE_URL_HERE'; const SUPABASE_KEY = 'YOUR_SUPABASE_ANON_KEY_HERE';
  1. Replace them with your actual credentials (keep the quotes!):
const SUPABASE_URL = 'https://xxxxx.supabase.co'; const SUPABASE_KEY = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...';
  1. Save the file
  2. Repeat steps 1-4 for mobile.html

Step 6: Host Your Widget

Option A - GitHub Pages (Recommended):

  1. Create a GitHub account at github.com
  2. Create a new repository (name it something like "my-todo-widget")
  3. Upload both HTML files (index.html and mobile.html)
  4. Go to Settings ‚Üí Pages
  5. Under "Source", select "main" branch
  6. Click Save
  7. Your widget will be at: https://your-username.github.io/my-todo-widget/

Option B - Local Testing:

  1. Just open the HTML files in your browser
  2. Works but won't be accessible from other devices

‚ùì Troubleshooting

Widget shows "Loading..." forever:

  • Check that you replaced BOTH URL and KEY in the code
  • Make sure you ran the SQL in Step 2
  • Check browser console (F12) for error messages

Tasks don't sync between devices:

  • Make sure you enabled Replication in Step 3
  • Check that both devices are using the same URL

"Add List" doesn't work:

  • Make sure you created BOTH tables in Step 2
  • Check the browser console for errors

Need help? Check your browser console (press F12) for error messages.
